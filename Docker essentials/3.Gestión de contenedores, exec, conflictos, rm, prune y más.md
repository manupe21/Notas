
Ejecutar comandos de forma interativa dentro del contenedor

````
docker exec <contenedor> <comando>

	-it: Abre un terminal interactivo con el contenedor
	-e: AÃ±ade una variable de entorno
````

Un contenedor a diferencia que en kubernetes se puede parar e iniciar

````
docker stop <contenedor>
````

Para iniciar un contenedor podemos usar el comando docker start

````
docker start <contenedor>

	-a: Enlaza la salida del contenedor (estandar y error) a nuestro terminal
	-i: de modo interativo, podemos interactuar con la terminal
````

Para copiar archivos de nuestro host a docker

````

docker cp <archivoHost> contenedor:/ruta
````




Para borrar todos los contenedores que tenemos parados podemos usar el comando *prune*

````
docker <container|image> prune
````

Se puede realizar un cambio de nombre de un contenedor con el comando docker rename

````
docker rename <nombreAntiguo> <nombreNuevo>
````
