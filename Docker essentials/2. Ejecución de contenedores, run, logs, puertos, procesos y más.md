
Comando para ejecutar un contenedor y correr un contenedor

````
docker run <image>
````

Ejecutar contenedores con `docker run` puede pasar de algo simple a algo muy complejo rápidamente. Aquí tienes los **flags** esenciales divididos por categorías para que los tengas siempre a mano.

Recuerda que el orden básico es: `docker run [FLAGS] IMAGEN [COMANDO]`.


Se puede ejecutar también un *docker create* si no quieres lanzar el contenedor, solo crearlo

---

## 1. Control de Interacción y Ciclo de Vida

Estos definen cómo te comunicas con el contenedor y qué pasa cuando termina.

|**Flag**|**Descripción**|**Uso común**|
|---|---|---|
|`-d`|**Detached:** Corre el contenedor en segundo plano.|Para bases de datos o servidores web.|
|`-it`|**Interactive + TTY:** Te permite interactuar con la terminal del contenedor.|Para entrar a una `bash` o `sh`.|
|`--rm`|**Remove:** Elimina el contenedor automáticamente al detenerse.|Para pruebas rápidas o scripts temporales.|
|`--name`|Asigna un **nombre** personalizado al contenedor.|Para no tener que usar el ID aleatorio.|

---

## 2. Puertos y Red (Networking)

Fundamental para que el mundo exterior (o tu máquina) llegue al servicio dentro del Docker.

- **`-p [puerto_host]:[puerto_container]`**: Mapea puertos.
    
    - Ejemplo: `-p 8080:80` (Ves el sitio en el puerto 8080 de tu PC, pero el servidor dentro corre en el 80).
        
- **`--network`**: Conecta el contenedor a una red específica.
    
    - Ejemplo: `--network mi-red-de-app`.
        

---

## 3. Volúmenes y Persistencia

Docker es "efímero" (lo que guardas dentro se borra al eliminar el contenedor). Los volúmenes evitan esto.

- **`-v [ruta_host]:[ruta_container]`**: Monta una carpeta de tu PC dentro del contenedor.
    
    - Ejemplo: `-v /mis/datos:/data`.
        
- **`--mount`**: Una versión más moderna y explícita de `-v`. Muy recomendada en entornos profesionales.
    

---

## 4. Variables de Entorno y Recursos

Para configurar la aplicación sin cambiar el código.

- **`-e`**: Define variables de entorno.
    
    - Ejemplo: `-e MYSQL_ROOT_PASSWORD=mi-password`.
        
- **`--env-file`**: Carga un archivo completo de variables (ideal para archivos `.env`).
    
- **`--restart`**: Define la política de reinicio.
    
    - `--restart always`: Si el servidor se apaga o Docker se reinicia, el contenedor sube solo.
        
- **`-m`**: Limita la **memoria RAM**.
    
    - Ejemplo: `-m 512m`.


Para revisar los logs de un contenedor podemos usar el siguiente comando


````
docker logs <contenedor> [-f En directo]
````


## Análisis de un contenedor

Para mostrar más detalles de un contenedor podemos lanzar el comando docker inspect

````
docker inspect <contenedor> 
# Este comando nos saca una salida con diversos detalles importantes como por ejemplo almacenamiento o red #
````
